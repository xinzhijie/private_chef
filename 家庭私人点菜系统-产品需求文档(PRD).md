# 家庭私人点菜系统\-产品需求文档\(PRD\)

## 1\. 项目概述

### 1\.1 项目简介

本系统为**纯前端家庭私人点菜系统**，基于 Vue3 \+ SQLite（sql\.js）开发，无需后端服务器、无需部署数据库，数据本地持久化存储。主要用于家庭日常早/中/晚餐菜品点餐、菜谱查看、菜品管理，支持多用户注册登录、角色权限区分，满足家庭成员共用、透明点餐、菜品详情查看、管理员维护菜品的核心需求。

### 1\.2 核心目标

- 实现家庭多人分账号独立登录，权限差异化管控

- 支持早、中、晚三餐菜品分类展示与点餐

- 所有点餐记录公开可见，菜品标注点餐用户，家庭点餐透明化

- 支持菜品详情查看（食材、备注、做法、图片）

- 管理员统一维护菜品（新增、编辑、上下架、删除）

- 所有数据本地SQLite持久化，重启不丢失，支持长期使用

### 1\.3 技术栈

- 前端框架：Vue3 \+ Vite

- 数据库：sql\.js（前端内置SQLite，纯前端无服务）

- 状态管理：Pinia

- UI组件：Element Plus

- 持久化：本地SQLite数据库文件存储

## 2\. 用户角色与权限体系（核心）

系统共设置**三种角色**，权限严格区分，所有用户均可查看全部菜品、全部点餐记录

|角色|查看菜谱|查看所有人点餐记录|自主点餐|菜品管理（增删改/上下架）|
|---|---|---|---|---|
|管理员 admin|? 全部|? 全部|? 支持|? 全部权限|
|普通用户 user|? 全部|? 全部（显示点餐人姓名）|? 支持|? 无权限|

## 3\. 整体功能模块

系统分为五大核心模块：账号权限模块、菜品管理模块、点餐模块、菜品详情模块、点餐记录模块

### 3\.1 账号权限模块

#### 3\.1\.1 用户注册

- 支持用户名、密码注册，用户名唯一不可重复

- 默认注册角色为 **普通用户\(user\)**

- 管理员可后台手动创建管理员、只读用户

#### 3\.1\.2 用户登录

- 账号密码校验登录，登录成功缓存用户信息

- 根据用户角色动态分配页面操作权限

- 未登录用户禁止访问所有功能页面，自动跳转登录页

#### 3\.1\.3 路由权限拦截

- 仅管理员可进入菜品管理后台页面

- 只读用户隐藏所有点餐操作按钮

### 3\.2 菜品管理模块（仅管理员）

- **新增菜品**：录入菜名、上传菜品图片、选择餐段（早/中/晚）、填写食材清单、菜品备注/做法

- **编辑菜品**：修改菜品所有信息、替换图片、更新食材和备注

- **上下架管理**：可手动设置菜品上架（前台展示可点餐）、下架（前台隐藏）

- **删除菜品**：永久删除指定菜品数据

### 3\.3 前台点餐模块（核心功能）

#### 3\.3\.1 餐段切换

前台支持顶部切换：早餐 / 午餐 / 晚餐，切换后实时展示对应餐段、已上架的菜品

#### 3\.3\.2 菜品展示

- 仅展示**已上架**菜品

- 展示菜品图片、菜品名称

- 点击菜品唤起详情弹窗

#### 3\.3\.3 点餐功能

- 管理员、普通用户可点击点餐按钮下单

- 只读用户无点餐按钮，无法下单

- 点餐自动记录：当前用户ID、用户名、菜品信息、点餐日期、餐段

#### 3\.3\.4 公共点餐展示

- 所有角色均可查看当日当前餐段的**全部用户点餐记录**

- 每条点餐记录标注**点餐用户名**，透明可查

- 可区分自己点餐、他人点餐（可做高亮区分）

### 3\.4 菜品详情模块

用户点击任意菜品，弹出详情弹窗，展示完整菜品信息：

- 菜品高清图片

- 菜品名称

- **食材清单**：管理员录入的菜品原材料

- **菜品备注/做法**：烹饪步骤、口味要求、食用小贴士等

### 3\.5 点餐记录模块

- 按日期、餐段筛选点餐记录

- 展示所有用户点餐数据，关联显示点餐人用户名

- 数据永久保存，可追溯历史点餐记录

## 4\. 数据库设计（SQLite）

系统共3张核心数据表，结构精简、适配所有业务场景

### 4\.1 用户表 users

```sql
CREATE TABLE users (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  username TEXT UNIQUE NOT NULL,
  password TEXT NOT NULL,
  role TEXT NOT NULL  -- admin / user / viewer
);
```

### 4\.2 菜品表 foods

```sql
CREATE TABLE foods (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  name TEXT NOT NULL,        -- 菜名
  image TEXT,                -- 菜品图片base64
  type TEXT NOT NULL,         -- 餐段：breakfast/lunch/dinner
  status INTEGER DEFAULT 1,  -- 1上架 0下架
  material TEXT,             -- 食材清单
  remark TEXT,                -- 菜品备注/做法
  create_time DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

### 4\.3 点餐表 orders

```sql
CREATE TABLE orders (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  user_id INTEGER NOT NULL,    -- 点餐用户ID
  food_id INTEGER NOT NULL,    -- 菜品ID
  food_name TEXT NOT NULL,     -- 菜品名称
  username TEXT NOT NULL,      -- 点餐用户名（展示用）
  type TEXT NOT NULL,          -- 餐段
  order_date TEXT NOT NULL,    -- 点餐日期 YYYY-MM-DD
  create_time DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

## 6\. 核心交互规则

- 每日点餐按日期隔离，默认展示当日点餐数据

- 所有点餐记录公开透明，全员可见，标注点餐人

- 菜品下架后，前台不再展示，无法点餐，历史记录保留

- 菜品详情弹窗为全局通用交互，所有角色均可查看

- 权限控制前端精准拦截，无权限用户隐藏对应操作入口

- **移动端适配**：页面自适应手机、平板、电脑端
